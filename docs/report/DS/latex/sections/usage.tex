\section{Usage Examples}

Below are some examples of interaction with the application, illustrating the main usage flows. When designing the navigation flows, the \textbf{three-click rule} was always taken into account, aiming to make the main features accessible in just a few steps.

\subsection{Explore the Platform}

A user who opens the platform can start exploring it. They can discover proposed events or search for them directly via the search bar. A dedicated explore page allows filtering events and searching for users and organizations.

\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{../public/storyboard/Storyboard-Explore.png}
  \caption{Storyboard: Exploring the platform}
  \label{fig:storyboard-explore}
\end{figure}

\subsection{Login and Registration}

After opening the application, a user can log in or register to access the additional features offered by the platform.

\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{../public/storyboard/Storyboard-Login.png}
  \caption{Storyboard: Login and registration}
  \label{fig:storyboard-login}
\end{figure}

\subsection{Create an Event}

An organization that has registered on the platform can create events through a form where all the relevant data is entered. It is also possible to temporarily create a draft of the event and continue editing it later.

\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{../public/storyboard/Storyboard-Create-Event.png}
  \caption{Storyboard: Creating an event}
  \label{fig:storyboard-create-event}
\end{figure}

\subsection{Attend an Event}

A registered user can purchase tickets for different events and view them later. The tickets contain a QR code that organizations can use to verify them.

\begin{figure}[H]
  \centering
  \includegraphics[width=\textwidth]{../public/storyboard/Storyboard-Buy-Ticket.png}
  \caption{Storyboard: Purchasing a ticket}
  \label{fig:storyboard-buy-ticket}
\end{figure}

\subsection{Review an Event}

After attending an event, a user can decide to leave a review. Once submitted, they cannot leave another review for the same event, but they can modify or delete it.
